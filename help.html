iPhone and iPad: fixing no-sound playback

iOS and iPadOS protect speakers until you explicitly allow a page to play sound. If Read‑Aloud looks like it is playing but you do not hear audio, work through this checklist in order.

  1. Flip the physical mute switch to the unmuted position. Some models still route Safari audio through the ringer mute toggle.
  2. Press the hardware volume up button twice while Safari is open to raise the media volume, not the ringer volume.
  3. Tap inside the page once and press the Start button again. Mobile Safari requires a user gesture before allowing speech synthesis.
  4. If you are using Guided Access or Focus modes, briefly exit them and retry; some profiles block web audio.
  5. Try another browser such as Chrome for iOS. It still uses Safari’s engine but sometimes refreshes the voice list when Safari stalls.

If the issue persists, reboot the device to clear the speech queue, then open read-aloud.com/app as the first tab and retest.

Empty or missing voice list

Read‑Aloud depends on voices provided by your operating system. When the dropdown appears empty, it usually means the browser could not load the list yet. Try these steps:

  • Reload the page after waiting five seconds. Some browsers populate the list only after the voiceschanged event fires.
  • Switch browsers. Chrome, Edge, and Safari expose more voices than some privacy or kiosk browsers.
  • Install additional system voices. On macOS, open System Settings → Accessibility → Spoken Content; on Windows, open Settings → Time & Language → Speech; on Android, open Settings → Text-to-speech output.
  • If you are on a managed school device, confirm that speech synthesis is allowed. Some admin profiles disable it.

The Voice Gallery explains how each platform exposes voices and how to add more. After installing a new voice, close and reopen the browser to refresh the list.

Using Read‑Aloud offline

Read‑Aloud can keep speaking while offline as long as the page is already loaded and the selected voice is stored locally. System voices on macOS, iOS, Windows, and ChromeOS usually work without connectivity after the first use. Cloud-only voices offered by some extensions or third-party engines will not show up here because the tool intentionally avoids external streaming for privacy reasons.

Before boarding a flight or heading into a low-signal area, open the tool, choose a voice, and press Start for a few seconds. That initial playback caches the voice. If you later see errors offline, shorten the pasted text and make sure your device is not killing the browser tab in battery-saver mode.

Performance tips for long text

Very long pastes can overwhelm the browser queue. Break documents into smaller sections—three to five minutes of listening time is a sweet spot—and press Resume between sections to keep your place. Lowering the speed to 0.9× can also improve stability on older devices.

  • Use headings as cut points. Copy one chapter or subheading at a time.
  • Close other tabs running audio or heavy scripts to free memory.
  • If timing is important, note the progress percentage when you pause so you can resume from a nearby paragraph.
  • For research, keep a notepad next to the player and jot timestamps when you hear something to revisit.

The Focus guide and Voice Selection guide include pacing templates that balance comprehension with speed.

Known limitations

Read‑Aloud runs entirely in your browser, so a few constraints are built-in:

  • No MP3 export: The Speech Synthesis API does not return audio files. If you need recordings, consider native screen recording with system audio while respecting copyright.
  • Sleep and lock screens: Laptops that go to sleep will pause playback. On mobile, keep the screen awake during long listens.
  • Tab focus: Some browsers throttle background tabs. Keep Read‑Aloud in the foreground for the most reliable timing.
  • Voice quality varies: The app lists what your device provides; premium voices require installing them at the OS level.

We track browser changes and update the About page with roadmap items when APIs evolve.

Frequently asked questions

Can I use Read‑Aloud for school?

Yes. The tool is designed to be classroom-friendly and does not require sign-ins. Still, avoid pasting private student data.

Does it work with PDFs?

Copy the text from the PDF and paste it into the tool. Scanned PDFs without selectable text need OCR before they can be spoken.

Is my text stored?

No. Everything runs locally in your browser. Analytics track page visits only, not the text itself.

Where can I get live help?

Send a message through the Contact page. Include your browser, device, and a short description of what happened so we can reproduce the issue.
