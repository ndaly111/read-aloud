Known limitations: Why MP3 export is hard in browser text‑to‑speech

Last updated: December 2025

Quick summary:
  • Read‑Aloud can make your browser speak text using system voices.
  • But browsers typically do not provide websites a clean “audio stream” from system voices to save as a file.
  • That’s why “Download MP3” is not a simple button when using the Web Speech API.

What Read‑Aloud uses today

  • Primary: your browser/OS voices via the Web Speech API.
  • Fallback: a small offline robotic voice (meSpeak) when system voices aren’t available.

The key point: the high-quality “natural” voices are implemented inside your operating system and exposed to the browser as a speak command — not as a downloadable audio buffer.

Why websites usually can’t export MP3 from system voices

  • No standard “give me the audio bytes.” The Web Speech API is designed to play audio, not hand websites an audio file.
  • System voice engines are OS-level. The browser asks the OS to speak; it doesn’t always receive the waveform back.
  • Capturing system audio is sensitive. Browsers avoid giving websites a way to record whatever your device is playing without explicit consent.
  • Cross-browser reliability. Even if a hack works in one browser, it tends to fail elsewhere.

Realistic MP3 export approaches (and the tradeoffs)

Approach Voice quality Privacy Notes
Client-side synthesis (offline voice) Robotic / basic Strong (local) Possible on GitHub Pages, but quality is typically lower than system voices. Also requires extra CPU for encoding and careful browser testing.
Cloud TTS (server returns MP3) Often excellent Depends (text leaves device) Technically straightforward and high quality, but requires ongoing cost controls and very clear disclosure.
Bring-your-own API key Excellent Depends (to provider) Offloads cost to the user but increases complexity and support burden.
OS-level recording Varies Local Users can record their device audio for their own text. This is outside Read‑Aloud and varies by OS.

What we won’t do without clear opt-in

Any future export feature should be explicit about privacy. If text must go to a server to produce a file, the page should say so in plain English before the user clicks “Convert.”

Copyright + rights reminder

Use text you have the rights to use. Avoid uploading/pasting copyrighted books or paywalled content into any conversion service.
